## 4. 아키텍처 리뷰

### 기술 스택

```
Framework : React Native 0.81 + Expo SDK 54
Language  : TypeScript 5.9
Routing   : Expo Router v6 (file-based, 20 routes)
State     : Zustand 5 (5 stores) + TanStack Query 5 (6 hooks)
Backend   : Supabase (Auth + PostgreSQL + Storage + Realtime)
Editor    : TenTap (native) / contentEditable (web)
Storage   : R2 (primary) / Supabase Storage (fallback)
Offline   : expo-sqlite + sync engine
Analytics : @sentry/react-native
```

```
[사용자 편집] → dirty 마킹
        ↓
[뒤로가기] → 변경사항 있으면 자동 저장 → 히스토리에 이전 버전 기록 → 이동
        ↓
[수동 저장] → 토스트 표시 → 히스토리에 이전 버전 기록
        ↓
[히스토리 복원] → 현재 내용도 히스토리에 보존 → 선택 버전으로 교체

용량 최적화:
- 5분 이내 재저장 → 마지막 히스토리에 병합 (중간 버전 방지)
- 노트당 최대 20개 히스토리
- 30일 초과 자동 삭제
```

## 10. 결론

> **"3/31 대비 완성도가 60% → 80%로 크게 향상되었다."**

어제 회의에서 지적된 6개 미해결 이슈가 **전부 해결**되었고, 추가로 **자동 저장 + 히스토리 복원**이라는 핵심 사용자 경험 개선이 이루어졌다.

**남은 것은 코드가 아니라 인프라 설정과 에셋이다.** Sentry DSN, R2 Worker 배포, 앱 아이콘만 준비되면 EAS Build → Play Store 제출이 가능하다.

**타임라인: 4/21 공개 출시 목표는 현실적이다.**
